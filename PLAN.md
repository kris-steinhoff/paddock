# paddock — implementation plan

## Context

The repo currently holds only `SPEC.md` (the agreed design). The goal is to build the first working version of `paddock`: a uv-managed Python CLI that resolves a repo's per-repo Dockerfile and merged settings from XDG config, then builds, runs, or removes the corresponding Docker image with configured environment variables (a GitLab PAT) injected. Scope is the three commands in the spec (`build`, `run [--build]`, `remove`) and nothing from the manual toil backlog.

## Project layout (uv + src layout)

```
pyproject.toml          # project metadata, deps, ruff/ty/pytest config
uv.lock                 # generated by uv
src/paddock/
  __init__.py
  cli.py                # typer app: build / run / remove
  resolver.py           # repo path -> Dockerfile path, image tag, settings file list
  config.py             # pydantic models, settings discovery+merge, env resolution
  docker.py             # thin subprocess wrappers around docker build/run/rmi/inspect
tests/
  conftest.py           # tmp XDG base fixture
  test_resolver.py
  test_config.py
  fixtures/             # sample repos/ tree for tests
```

## Modules

### `resolver.py`: path is the convention

- `config_base() -> Path`: `${XDG_CONFIG_HOME:-~/.config}/paddock`.
- `parse_repo(repo: str) -> list[str]`: strip surrounding slashes, split on `/`, reject empty segments and too-few segments (need at least host + name).
- `RepoTarget` dataclass holding: `segments`, `repos_dir` (`base/repos`), `leaf_dir` (`repos_dir/<segments...>`), `dockerfile` (`leaf_dir/Dockerfile`), `image` tag, and `settings_paths` (ordered least to most specific).
- `image_tag(segments) -> str`: `"paddock/" + "/".join(segments)`, lowercased (Docker requires lowercase repository names).
- `settings_paths(segments) -> list[Path]`: `repos_dir/settings.yaml`, then one `settings.yaml` at each successive directory level down to the leaf, limited to those that exist.

### `config.py`: merged settings

- Pydantic models: `CommandValue(BaseModel)` with required `command: str`. `EnvValue = str | CommandValue`. `Settings(BaseModel)` with `environment: dict[str, EnvValue] = {}` (`extra="forbid"` to catch typos).
- `load_merged(paths) -> Settings`: read each YAML (`yaml.safe_load`), deep-merge the top-level hash in order (most specific wins per `environment` key), then validate the merged dict into `Settings`. Surfaces parse/validation errors with the offending file path.
- `resolve_environment(settings) -> dict[str, str]`: for each entry, a string is used as-is. A `CommandValue` runs `subprocess.run(["sh", "-c", cmd], capture_output=True, text=True, check=True)` and uses `stdout` with the trailing newline stripped (`rstrip("\n")`). Non-zero exit raises and aborts the command.

### `docker.py`: subprocess wrappers

List-form `subprocess.run` (no shell), so env values pass safely:

- `build(image, context_dir)`: `docker build -t <image> <context_dir>`.
- `run(image, env)`: `docker run -it --rm` + a `-e KEY=VALUE` pair per env entry + `<image>`.
- `image_exists(image) -> bool`: `docker image inspect <image>` (returncode check, output suppressed).
- `remove(image)`: `docker rmi <image>`.

### `cli.py`: typer app

- `app = typer.Typer(no_args_is_help=True)`, exposed as the `paddock` console script.
- `build(repo)`: resolve target, error if `dockerfile` missing, `docker.build`.
- `run(repo, build: bool = typer.Option(False, "--build"))`: if `--build`, build first. Otherwise error (with a hint to pass `--build`) when `image_exists` is false. Then `load_merged` + `resolve_environment` + `docker.run`.
- `remove(repo)`: resolve target, `docker.remove`.
- Errors raise `typer.Exit(code=1)` after printing a clear message (missing Dockerfile, unbuilt image, bad settings, failed command, docker failure).

## Tooling config (`pyproject.toml`)

- `[project]` deps: `typer`, `pydantic>=2`, `pyyaml`. `requires-python = ">=3.10"`.
- `[project.scripts] paddock = "paddock.cli:app"`.
- `[dependency-groups] dev = ["pytest"]` (uv-native dev group).
- Build backend: `hatchling` with `packages = ["src/paddock"]`.
- `[tool.ruff]` formatting + lint (sensible default select), `[tool.ty]` minimal, `[tool.pytest.ini_options] testpaths = ["tests"]`.
- Bootstrap with `uv sync`, generating `uv.lock` and `.venv`.

## Tests (pytest, no Docker needed)

Pure logic only. `docker.py` is kept thin and not exercised against a real daemon.

- `test_resolver.py`: repo parsing (incl. arbitrary depth like `gitlab.com/org/subgroup/project`), image tag lowercasing, Dockerfile path, and the ordered settings-paths list (monkeypatch `XDG_CONFIG_HOME` to `tmp_path`, lay down a fixture tree).
- `test_config.py`: deep-merge precedence (org overrides provider overrides global), string vs `command` env values (`command: echo hi` -> `hi`), non-zero command aborts, `extra="forbid"` rejects unknown keys.

## Verification

1. `uv sync` then `uv run paddock --help` shows `build`, `run`, `remove`.
2. `uv run ruff format --check . && uv run ruff check . && uv run ty check && uv run pytest`.
3. End-to-end smoke test with a temporary config:
   - Point `XDG_CONFIG_HOME` at a temp dir, create `repos/github.com/kris-steinhoff/paddock/Dockerfile` (e.g. `FROM alpine` + `CMD env`) and a `settings.yaml` with `environment: { GITLAB_PAT: "test" }`.
   - `uv run paddock run github.com/kris-steinhoff/paddock --build` builds the image and runs it, with `GITLAB_PAT=test` visible in the container's `env` output.
   - `uv run paddock remove github.com/kris-steinhoff/paddock` deletes the image.
